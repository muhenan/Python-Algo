# 搜索算法讨论

这份笔记主要讨论刷题中最常见的几类搜索算法：`BFS`、`DFS`、`回溯`、`Dijkstra`。重点不是背定义，而是判断：

- 题目为什么适合某种搜索
- 某个搜索为什么会超时
- 能不能从图论和复杂度上解释这种差异

---

## 1. 先把问题抽象成图

很多搜索题本质上都可以抽象成图：

- 状态 = 图上的节点
- 一次合法操作 = 一条边
- 从起点状态走到目标状态 = 搜索过程

例如 `LeetCode 1091 - Shortest Path in Binary Matrix`：

- 每个值为 `0` 的格子可以看成一个节点
- 两个格子如果 8 个方向相邻，就有一条边
- 从左上角走到右下角，就是从起点节点走到终点节点
- 每走一步代价都相同，都是 `1`

于是这题就变成了：

> 在无权图中，求起点到终点的最短路长度。

这一步抽象非常关键。很多题一旦能抽象成图，选算法就清楚很多。

---

## 2. 为什么 1091 里 BFS 比 DFS 更好

### 2.1 BFS 的本质：按层扩展

`BFS` 会一层一层向外扩展：

- 第 `0` 层：起点
- 第 `1` 层：一步能到的所有点
- 第 `2` 层：两步能到的所有点
- ...

所以在无权图里，`BFS` 第一次到达某个点时，走过的步数就是最短步数。

设起点为 `s`，定义 `dist(v)` 表示从 `s` 到 `v` 的最短路径长度。

`BFS` 的核心结论是：

> 第一次访问到节点 `v` 时，当前层数恰好等于 `dist(v)`。

### 2.2 简短证明

假设 `BFS` 现在访问到了第 `k` 层的某个节点 `v`。

1. `v` 一定是从第 `k - 1` 层某个节点扩展出来的，所以存在一条长度为 `k` 的路径到 `v``
2. 因此 `dist(v) <= k`
3. 如果 `dist(v) < k`，说明存在更短路径，那么 `v` 应该更早就在前面的层中被访问到
4. 这与“现在才第一次访问到 `v`”矛盾

所以只能有：

> `dist(v) = k`

因此，在无权图中：

> `BFS` 第一次到达终点时，得到的就是最短路。

这就是为什么像 `1091` 这种题，`BFS` 可以在遇到终点时直接返回答案。

---

## 3. 为什么 DFS 在最短路问题里容易超时

### 3.1 DFS 的本质：先一条路走到底

`DFS` 的搜索顺序不是按层，而是：

- 先顺着一个方向不断往深处走
- 走不动了再回溯
- 再尝试别的分支

所以 `DFS` 第一次到达终点时，只能说明：

> 找到了一条可行路径

但它**不能说明这是最短路径**。

这意味着如果用 `DFS` 做最短路，通常就得：

- 枚举很多可能路径
- 对所有路径取最小值
- 再配合剪枝尽量减少搜索

### 3.2 为什么会指数爆炸

设：

- 每个状态平均有 `b` 个分支
- 搜索深度上界为 `m`
- 最短解深度为 `d`

那么：

- `BFS` 找最短路时，通常只需要搜索到第 `d` 层  
  复杂度约为 `O(1 + b + b^2 + ... + b^d) = O(b^d)`

- `DFS` 若要保证找到最短路，最坏情况下要枚举大量深层路径  
  复杂度可能接近 `O(b^m)`

当 `m >> d` 时，差距是指数级的。

### 3.3 1091 为什么特别容易让 DFS 超时

在 `1091` 里：

- 每个格子最多可以往 `8` 个方向走
- 图中可能存在大量不同路径
- `DFS` 虽然用 `visited` 防止成环，但仍会枚举大量简单路径
- 剪枝 `pathLength >= ans` 只有在先找到一个较优答案后才会明显生效

所以它慢不是实现细节导致的，而是算法本质决定的。

## 4. 什么时候 BFS 比 DFS 更合适

通常满足下面这些条件时，优先考虑 `BFS`：

1. 图是**无权图**
2. 或者每条边权都相同
3. 问的是**最少步数**、**最短边数**、**最少操作次数**
4. 目标是“第一次到达目标就可以结束”

典型关键词：

- shortest path
- minimum steps
- minimum moves
- least number of operations
- 从起点到终点的最短距离

这类题本质上都是：

> 用层序扩展来保证第一次到达就是最优。

---

## 5. 什么时候 DFS 更合适

通常满足下面这些条件时，优先考虑 `DFS` 或回溯：

1. 题目要求找**任意一个可行解**
2. 题目要求**枚举所有方案**
3. 题目要求输出所有路径、所有组合、所有排列
4. 状态空间天然具有“选 / 不选 / 继续尝试”的递归结构

典型例子：

- 子集
- 组合
- 全排列
- 单词搜索
- N 皇后
- 迷宫中的所有路径

这类题的重点不是“最短”，而是“遍历状态空间”。

---

## 6. BFS、DFS、回溯之间的关系

很多时候 `DFS` 和 `回溯` 很像，但关注点略有不同：

### BFS

- 关注“层”
- 常用于最短路、最少步数
- 队列驱动

### DFS

- 关注“先深入，再回退”
- 常用于连通性、可达性、拓扑探索
- 递归或栈驱动

### 回溯

- 本质上通常是 `DFS`
- 但更强调“做选择 -> 递归 -> 撤销选择”
- 常用于枚举全部解、搜索组合空间

可以粗略理解为：

> 回溯 = 带有选择树语义的 DFS

---

## 7. 什么时候该用 Dijkstra

如果图不是无权图，而是：

- 每条边有不同代价
- 问“最小总代价”
- 问“最短加权路径”

这时普通 `BFS` 就不够了。

因为 `BFS` 保证的是：

> 边数最少

但题目要的可能是：

> 权重和最小

例如：

- 一条路径走了 2 步，总权重 100
- 另一条路径走了 3 步，总权重 5

这时边数更少不代表代价更小。

当边权都是非负数时，通常应该使用 `Dijkstra`。

### 一句话区分

- 无权最短路：`BFS`
- 非负权最短路：`Dijkstra`
- 负权图最短路：通常考虑 `Bellman-Ford` / `SPFA` 等

## 8. 数学上的判断准则

可以记下面这个判断方式。

### 情况 A：无权图，目标是最短路

设图为 `G = (V, E)`，边权全部相同。

目标是求从起点 `s` 到终点 `t` 的最短边数：

> `min number of edges from s to t`

那么 `BFS` 具有严格正确性保证，因为它按距离层序遍历，首次访问节点时就已经确定最短距离。

这时 `DFS` 若不枚举足够多路径，无法保证正确；若想保证正确，最坏情况下又会退化到路径枚举，代价更高。

### 情况 B：只要求可达，不要求最短

若只需要判断：

- 能不能到达
- 连通块有哪些
- 某个区域有多大

那么 `BFS` 和 `DFS` 都可以，时间复杂度通常都是 `O(V + E)`。

这时谁更好，更多取决于：

- 代码风格
- 递归深度风险
- 是否需要按层信息

### 情况 C：要求枚举全部解

若题目目标是列出所有解，那么 `BFS` 通常没有优势，`DFS / 回溯` 更自然。

因为这时搜索重点是：

> 完整遍历搜索树

而不是最早找到一个最优答案。

---

## 9. 刷题时的快速判断模板

看到搜索题时，可以先问自己下面几个问题。

### 1. 状态是什么？

- 一个格子？
- 一个字符串？
- 一个数组位置？
- 一个图节点？

### 2. 一步转移是什么？

- 上下左右移动？
- 改变一个字符？
- 选择一个数字？

### 3. 问题要的是什么？

- 任意一个解？
- 所有解？
- 最少步数？
- 最小总代价？

### 4. 按这个表选算法

| 题目目标 | 常用算法 |
|---|---|
| 找任意可行解 | DFS |
| 枚举所有解 | DFS / 回溯 |
| 无权最短路 / 最少步数 | BFS |
| 非负权最短路 | Dijkstra |
| 连通性 / 染色 / 遍历区域 | BFS 或 DFS |

---

## 10. 针对 1091 的最终结论

`LeetCode 1091 - Shortest Path in Binary Matrix` 的关键特征是：

- 图是无权图
- 每次移动代价相同
- 目标是最短步数

因此：

> 这题应该优先用 `BFS`。

`DFS` 可以写成对照版本帮助理解，但它在本题中不是合适的主解法，因为它需要枚举大量路径，最坏情况下复杂度会远高于 `BFS`。

---

## 11. 一句话总结

- **无权图最短路，优先想 `BFS`**
- **枚举所有方案，优先想 `DFS / 回溯`**
- **带权最短路，优先想 `Dijkstra`**

如果一道题做不出来，先不要急着写代码，先问：

> 这题到底是在找最短路，还是在枚举状态空间？

这个问题一旦想清楚，算法通常也就选对了。
